## 📊 데이터 파이프라인 검토 및 A/B 테스트 자동화 요구사항 정의

### 1. 데이터 파이프라인 구조 검토 (Review of Data Pipeline Structure)

**현재 구조:**
코다리 에이전트가 설계한 `data_loader.py`는 `AVD -> CTR -> CR`의 핵심 KPI 흐름을 위한 초기 로더 클래스를 정의하고 있습니다. 이 구조는 광고 캠페인의 성과를 측정하는 데 필요한 **핵심 지표(Metric)**와 그 **데이터 소스** 간의 연결 고리를 설정합니다.

**전략적 평가:**
현재 구조는 데이터 수집 및 정제(Load & Process)의 기초를 성공적으로 설계했습니다. 하지만 A/B 테스트 자동화라는 목표를 달성하기 위해서는 이 흐름에 다음 두 가지 핵심 요소가 추가되어야 합니다.

1.  **실시간 연동 레이어 (Real-time Linking Layer):** 광고 플랫폼 데이터와 내부 KPI 데이터를 실시간으로 매핑할 수 있는 구조.
2.  **결정론적 변수 관리 (Deterministic Variable Management):** 어떤 광고 그룹(A/B)이 어떤 KPI에 영향을 미쳤는지 명확히 추적하고, 이 결과를 기반으로 다음 행동을 결정하는 로직.

### 2. A/B 테스트 데이터 입력 및 자동화 스크립트 요구사항 정의 (Requirements for Automation)

수익화 모델 검증과 경쟁사 ROI 분석 목표를 달성하기 위해, 광고 캠페인 세팅부터 성과 분석까지의 과정을 자동화할 수 있는 시스템을 구축해야 합니다.

#### A. 데이터 입력 요구사항 (Data Input Requirements)

| 단계 | 요구 데이터 항목 | 출처/형식 | 목적 |
| :--- | :--- | :--- | :--- |
| **1. 캠페인 설정** | 광고 플랫폼 ID, 타겟 그룹 정의 (A그룹, B그룹), 예산 할당액 | 외부 API 연동 또는 수동 입력 폼 | 테스트 조건 명확화 |
| **2. 트래픽 데이터** | 각 그룹별 유입량(Impression/Click), 세션 데이터 | 광고 플랫폼 API 추출 | 기본 성과 측정 기반 마련 |
| **3. 행동 지표 (KPI)** | AVD, CTR, CR 데이터 (데이터 로더를 통해 정제된 값) | `data_loader.py`에서 파이프라인을 거쳐 입력 | 최종 전환 효율 측정 |
| **4. 비용 데이터** | 각 그룹별 광고 비용(Spend), ROAS 계산에 필요한 지출액 | 광고 플랫폼 API 추출 | 수익성 분석의 핵심 |

#### B. 자동화 스크립트 요구사항 (Automation Script Requirements)

1.  **데이터 수집 모듈 (`data_loader` 확장):**
    *   **요구사항:** `data_loader.py`를 확장하여, 외부 광고 API(예: Google Ads, Meta API 등)에서 데이터를 직접 가져오는 함수(`fetch_ad_performance`)를 구현해야 합니다.
    *   **기능:** 특정 캠페인 ID와 기간을 입력하면, 해당 기간 동안의 Raw 데이터(Impression, Click, Spend)를 자동으로 추출하여 DB 또는 파일 형태로 저장해야 합니다.

2.  **KPI 연계 및 분석 스크립트 (`analyzer` 모듈):**
    *   **요구사항:** `business.md`에 정의된 **KPI 연계 매핑 테이블**을 기반으로, 수집된 Raw 데이터와 정제된 KPI(AVD, CTR, CR)를 결합하는 통합 분석 함수(`calculate_roi`)를 작성해야 합니다.
    *   **기능:** A그룹과 B그룹의 데이터를 입력받아, 단순 전환율 비교뿐만 아니라 **비용 대비 성과(ROAS)** 측면에서 두 그룹의 효율성을 통계적으로 비교하고, 어떤 후크/비주얼 조합이 가장 높은 ROI를 창출했는지 계산해야 합니다.

3.  **결정 및 보고 자동화 (`decision_engine`):**
    *   **요구사항:** 분석 결과를 바탕으로 **경험 기반의 권고(Recommendation)**를 도출하는 로직을 포함해야 합니다.
    *   **기능:** ROI가 특정 임계값(예: B 그룹이 A 그룹보다 15% 이상 높음)을 초과할 경우, 시스템이 자동으로 "비주얼 스타일 X는 중단하고 Y 스타일로 전환하라"와 같은 구체적인 경영적 권고를 생성해야 합니다.
