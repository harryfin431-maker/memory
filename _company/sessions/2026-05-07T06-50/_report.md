## 💎 Visual Component Specs 및 에셋 요구사항 (Module_Upgrade) 최종 산출물

본 문서는 Developer가 '고급 시스템 모듈 업그레이드' 기능 구현에 필요한 시각적 컴포넌트와 에셋을 제작할 수 있도록 Deep Navy/Neon Cyan 스타일을 완벽히 반영하여 제공합니다.

### 1. 브랜드 비주얼 시스템 요약 (Master System Blueprint)

*   **핵심 컬러 팔레트:**
    *   Primary Background/Base: **Deep Navy Blue** (`#0A1931`) - 신뢰감, 깊이
    *   Accent/Highlight: **Neon Cyan** (`#00FFFF`) - 미래지향적, 에너지
    *   Secondary Text/Detail: Light Gray (`#CCD6F6`) - 가독성 확보
    *   CTA/Action Background (Subtle): Deep Navy Blue with slight cyan glow effect.

*   **타이포그래피 시스템:**
    *   Headlines (H1, H2): **Montserrat Bold** (`font-weight: 800`, `font-size: 3rem` 이상) - 강렬하고 구조적인 느낌
    *   Body Text/Descriptions: **Inter Regular** (`font-weight: 400`, `font-size: 1.125rem`) - 높은 가독성 확보
    *   Code/Data Display: **Space Mono** (`font-family: monospace`) - 시스템적 느낌 강조

### 2. Visual Component Specifications (핵심 컴포넌트)

#### A. 모듈 카드 (Module Card - 핵심 시각 요소)

| 속성 | 상세 사양 | 적용 이유 |
| :--- | :--- | :--- |
| **Container Background** | Deep Navy Blue (`#0A1931`) | 시스템의 기반과 신뢰도 표현 |
| **Border/Accent** | 1px Neon Cyan (`#00FFFF`) 테두리 및 하단 라인 | 에너지와 구분을 통한 강조 |
| **Text Color** | Light Gray (`#CCD6F6`) | 배경과의 대비를 통한 안정적인 가독성 |
| **Hover Effect** | 마우스 오버 시, 전체 배경이 미세하게 밝아지며 Cyan Glow 효과 적용 (Transition: 0.3s ease) | 상호작용 및 고급스러움 부여 |
| **Shadow** | Subtle Inner Shadow (Deep Navy로 깊이감 표현) | 입체감 유지 |

#### B. CTA 버튼 (Call to Action Button)

*   **Primary CTA (고가치 유도):**
    *   Background: Neon Cyan (`#00FFFF`)
    *   Text Color: Deep Navy Blue (`#0A1931`)
    *   Font Weight: Bold
    *   Border Radius: 8px (모서리를 부드럽게)
    *   **Glow Effect:** 버튼 주변에 미세한 Cyan 발광 효과 추가.

*   **Secondary CTA (보조):**
    *   Background: Transparent / Deep Navy Blue with Neon Cyan Outline
    *   Text Color: Neon Cyan
    *   Font Weight: Medium

### 3. Asset Requirements (필수 에셋 준비 목록)

Developer는 아래 사양에 맞춰 다음 에셋들을 제작해야 합니다. 모든 에셋은 **Deep Navy/Neon Cyan** 팔레트를 엄격하게 준수해야 합니다.

1.  **Module Card Component:** 위에서 정의된 A 항목의 Figma 컴포넌트 파일 (State: Default, Hover)
2.  **CTA Button Component:** 위에서 정의된 B 항목의 Figma 컴포넌트 파일 (Primary/Secondary State)
3.  **Data Visualization Mockup:** Module_Upgrade 기능에 따른 KPI(매출, 유지율 등)를 시각화하는 그래프/차트 디자인 템플릿 (Deep Navy 배경에 Cyan 라인 강조)
4.  **System Diagram Template:** 시스템 프레임워크(`Methodology`)를 시각화하는 다이어그램 템플릿 (노드와 연결선에 Neon Cyan 사용)
