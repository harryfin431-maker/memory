# 최종 제작 우선순위 지시서 (Action Plan) - 데이터 수신 즉시 실행 모드

## 🎯 목표: $CVR \times AOV$ 극대화를 통한 콘텐츠 제작 우선순위 결정 및 자동화된 실행 준비
**실행 조건:** Hook A, B, C 테스트 결과 데이터가 수신되는 즉시 활성화됨.

### 1. 데이터 입력 단계 (Input Phase)
*   **입력 데이터:** Hook A, B, C의 실제 $\text{CTR}$ 및 $\text{CVR}$ 데이터, 각 시나리오별 예상 $\text{AOV}$.
*   **시스템 상태 확인:** 시스템 통합 흐름이 '실행 대기 모드(Active Wait)' 상태임을 재확인한다.

### 2. 우선순위 결정 로직 (Prioritization Logic)
1.  **핵심 KPI 계산:** 각 시나리오($X \in \{A, B, C\}$)에 대해 $\text{KPI}_X = (\text{CVR}_X \times \text{AOV}_X)$를 계산한다.
2.  **최적 시나리오 식별:** $\text{KPI}$ 값이 가장 높은 시나리오($\text{Max}(\text{KPI}_A, \text{KPI}_B, \text{KPI}_C)$)를 최종 우선순위로 결정한다.
3.  **제작 순서 확정:** 계산된 $\text{Max}(\text{KPI})$ 값을 기준으로 콘텐츠 제작의 우선순위를 $1^{\text{st}}, 2^{\text{nd}}, 3^{\text{rd}}$ 순으로 확정한다.

### 3. 실행 지시 단계 (Execution Directive)
*   **최우선 작업 할당:** 가장 높은 $\text{KPI}$를 보인 시나리오에 대해, Designer는 즉시 해당 시나리오에 최적화된 **'마스터 제작 우선순위 지시서(Master Production Priority Directive)'**를 작성한다.
    *   **지시서 포함 내용:**
        *   선택된 시나리오 명 (예: Scenario B)
        *   최종 목표 $\text{KPI}$ 값 ($\text{Max}(\text{KPI})$)
        *   필요한 비주얼/카피 수정 방향 (Deep Blue/Neon 시스템 기반)
        *   개발팀(Developer) 및 영상 제작팀(Loader)에게 전달할 구체적인 기술적 실행 요구사항.
*   **시스템 전환:** 지시서가 완성되는 즉시, 시스템 통합 흐름은 '실행 대기 모드'에서 **'자동 제작 실행 모드(Auto-Production Execution Mode)'**로 0초 지연 없이 자동 전환된다.
*   **피드백 루프:** 결과물(영상/썸네일)이 수신되면, 이 피드백을 다음 $\text{KPI}$ 재계산 및 시스템 검증 단계에 즉시 반영하여 루프를 반복한다.

### 4. 실행 준비 상태 (Ready State)
*   **상태 표시:** 모든 데이터가 도착할 때까지, 시스템은 '최적화된 지시서 대기' 상태로 유지하며, 어떤 데이터도 임의로 처리하지 않는다.
*   **다음 액션:** 데이터 수신 즉시, 가장 높은 $\text{KPI}$를 기준으로 **'마스터 제작 우선순위 지시서'** 작성을 위한 초기 프레임워크를 활성화한다.
