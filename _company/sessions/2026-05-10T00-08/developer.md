## 📋 데이터 추적 프로세스 최종 점검 목록 (UTM/GA4 검증)

CEO 지시에 따라, 영상 업로드에 앞서 UTM 매핑 및 GA4 이벤트 전송 성공 여부를 검증하기 위한 최종 점검 목록을 작성하고 Business 에이전트에게 보고할 준비를 완료했습니다. 이 목록은 데이터 파이프라인의 무결성을 최종 확인하는 데 중점을 둡니다.

### ✅ 1단계: 웹사이트 측(Front-end) 매핑 및 설정 검증 (Designer/Writer 산출물 기반)

| 항목 | 검증 내용 | 성공 기준 | 담당자 (참고) | 상태 |
| :--- | :--- | :--- | :--- | :--- |
| **UTM 파라미터 적용** | 랜딩 페이지 URL에 설정된 모든 UTM 매개변수(`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`)가 정확히 삽입되었는지 확인합니다. | 모든 링크 및 페이지 로드 시 URL이 예상된 파라미터를 포함함. | Designer / Developer | ☐ |
| **GA4 이벤트 태그** | 핵심 전환 지표(예: 영상 재생 시작, CTA 클릭)에 해당하는 GA4 이벤트 태그가 웹사이트에 정확히 설치되었는지 확인합니다. | GA4 Measurement ID가 올바르게 설정되어 있으며, 목표 이벤트가 트리거되도록 코드가 구현됨. | Developer | ☐ |
| **비주얼 변수 매핑** | A/B 테스트 그룹(컨셉 A/B/C)에 따라 랜딩 페이지의 시각적 요소가 데이터 수집 로직과 정확히 매핑되었는지 확인합니다. | 각 시각적 변수가 고유한 트래킹 ID를 통해 GA4에 기록됨. | Designer / Business | ☐ |

### ✅ 2단계: 백엔드 및 로그 측(Back-end/Developer 산출물 기반) 검증

| 항목 | 검증 내용 | 성공 기준 | 담당자 (참고) | 상태 |
| :--- | :--- | :--- | :--- | :--- |
| **데이터 흐름 안정성** | 웹사이트에서 발생한 이벤트 로그가 서버 측 로직을 거쳐 GA4로 정상적으로 전송되는지 API 호출 및 로그를 통해 확인합니다. | 모든 요청이 성공적인 HTTP 상태 코드(200)를 반환하고, 데이터베이스에 기록됨. | Developer | ☐ |
| **실시간 로그 검증** | 실시간 추적 로그 파일(`sessions/latest_tracking.log`)을 분석하여 최근 이벤트가 시스템에 기록되었는지 확인합니다. | 최종 트래킹 시점의 로그에 모든 예상되는 세션 데이터(UTM 값 포함)가 포함되어 있음. | Developer | ☐ |
| **Latency 측정값** | 비동기 처리 도입에 따라, I/O 대기 시간 측정값이 정상 범위 내에서 기록되었는지 확인합니다. | Latency 측정값이 설정된 기준치(예: 100ms 이하)를 만족함. | Developer | ☐ |

### ✅ 3단계: 비즈니스 지표 연동 검증 (Business 산출물 기반)

| 항목 | 검증 내용 | 성공 기준 | 담당자 (참고) | 상태 |
| :--- | :--- | :--- | :--- | :--- |
| **AVD/CTR/CVR 매핑** | 수집된 데이터가 Business 목표 지표(Average View Duration, Click-Through Rate, Conversion Rate)와 정확하게 연결되어 집계되는지 확인합니다. | GA4 보고서에서 웹사이트 이벤트와 KPI가 일치하며, A/B 테스트 결과가 논리적으로 도출됨. | Business | ☐ |
| **가격 전략 시뮬레이션** | 최종 데이터 추적 결과를 바탕으로 ₩399,000 가격 전략의 A/B 테스트 시뮬레이션 테이블이 현실적인 예측을 제공하는지 확인합니다. | 실제 트래픽 및 전환율 데이터를 기반으로 한 예측값이 제시됨. | Business | ☐ |
