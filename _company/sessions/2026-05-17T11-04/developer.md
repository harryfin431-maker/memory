### ⚙️ 자동화 스크립트 구현 계획 (Video Production Pipeline)

**목표:** 영상 제작 파이프라인의 기술적 병목 현상을 최소화하고, 콘텐츠 생성부터 최종 에셋 전달까지의 안정적인 흐름을 구축합니다.

#### 1. 시스템 아키텍처 설계: 모니터링 및 제어 레이어 정의 (The Control Layer)
시스템 안정성을 확보하기 위해 모든 단계를 추적하고 병목 지점을 감지하는 중앙 제어 로직을 설계합니다.

*   **모듈 정의:** `Pipeline_Orchestrator`, `Asset_Validator`, `Workflow_Monitor` 세 가지 핵심 모듈로 분리합니다.
*   **상태 관리 (State Management):** 각 단계(스크립트 → Shot List → 디자인 → 썸네일)의 완료 상태를 JSON 또는 YAML 기반으로 저장하는 중앙 데이터베이스/파일 시스템을 정의합니다. 이는 Git 워크플로우와 연동하여 변경 사항을 추적하게 합니다.
*   **환경 변수 통합:** 모든 외부 API 호출(만약 사용한다면) 및 리소스 할당은 환경 변수를 통해 이루어지도록 설정하여, 키 노출 없이 안전성을 확보합니다 (회사 원칙 준수).

#### 2. 핵심 자동화 스크립트 구현 계획 (The Execution Layer)
가장 시간 잡아먹는 수동 작업(Writer/Designer의 결과물 통합 및 최종 패키징)을 대체하는 스크립트를 구현합니다.

*   **스크립트 A: Shot List → Asset Mapping (데이터 파이프라인)**
    *   **목표:** Writer의 스크립트와 Designer의 Shot List를 입력받아, 필요한 시각 요소(Shot List)가 영상 편집 지침(Script)과 정확히 매핑되도록 검증하는 스크립트를 구현합니다.
    *   **기능:** Shot List 항목별로 요구되는 비주얼 키워드(`Deep Blue/Neon Cyan` 적용 범위, 프레임 길이 등)를 자동으로 추출하여 편집 지침 파일에 삽입합니다.
*   **스크립트 B: Visual Consistency Checker (자동 검증 루프)**
    *   **목표:** Designer가 정의한 'The Flow Principle' 및 'Deep Blue/Neon Cyan' 가이드라인이 최종 썸네일 에셋(`designer_final_shorts_thumbnail_assets.md`)에 100% 반영되었는지 확인하는 검증 루프를 구현합니다.
    *   **기능:** 최종 디자인 파일의 메타데이터(색상, 레이아웃 구도)와 정의된 규칙을 비교하여 불일치 시 에러를 발생시키고 수동 개입을 요청합니다 (Self-Verification Loop).
*   **스크립트 C: Delivery Package Generator (최종 패키징)**
    *   **목표:** 영상 제작에 필요한 모든 요소(편집 지침, 썸네일 에셋, 제목/설명/해시태그)를 하나의 최종 패키지(ZIP 또는 MD 파일)로 자동 압축하여 제공합니다.

#### 3. 기술적 병목 최소화 전략 (Stability & Performance)
스크립트 실행 중 발생할 수 있는 오류와 지연을 관리합니다.

*   **병렬 처리 도입:** Shot List 기반의 에셋 생성 및 검증 작업은 병렬로 처리될 수 있도록 설계하여 전체 시간을 단축합니다. (`multiprocessing` 또는 `asyncio` 활용 고려)
*   **에러 핸들링 강화:** 각 단계별 실패 지점(예: 이미지 포맷 오류, 텍스트 길이 초과)에서 즉시 중단하고 상세한 로그를 기록하는 예외 처리 로직을 모든 스크립트에 내재화합니다. (`try-except` 블록의 세밀화).
*   **Git 연동:** 모든 자동화 스크립트 실행 결과와 중간 산출물은 Git에 커밋하여 버전 관리를 철저히 합니다.
