CEO님의 지시사항을 확인했습니다. Designer가 확정한 DCI 기준과 Business의 KPI 연계 로직을 데이터 모델(API 명세서)에 반영하여, 시각적 일관성 지표(DCI)를 측정하고 연결할 수 있는 최종 필드를 추가하도록 개발팀에 전달하고 구현 계획을 확정하겠습니다.

이는 경험의 깊이(Wisdom)와 AI 효율성(Efficiency)을 시스템적으로 증명하는 핵심 단계입니다. 추측 없이 명확한 요구사항을 정의하여, 개발팀이 모호함 없이 바로 작업할 수 있도록 하겠습니다.

요구사항에 따라 백엔드 API 명세서에 반영되어야 할 최종 요구사항과 구현 계획을 아래와 같이 정리했습니다.

### 📝 API 명세서 업데이트 요구사항 (최종)

**대상:** 백엔드 API 명세서 및 데이터 모델 (`Subscription`, `BundleOption`, `TransactionLog` 등 관련 엔티티)
**목표:** 시각적 일관성(DCI) 지표와 핵심 성과 지표(KPI)를 직접 연결하는 필드를 추가하여, 데이터 기반의 효율성 측정 구조를 확립합니다.

#### 1. 신규 데이터 모델 필드 정의 (추가 항목)

기존 데이터 모델에 다음 두 가지 유형의 필드를 필수적으로 추가해야 합니다.

**A. 시각적 일관성 지표 (DCI Linkage)**
*   **필드명:** `visual_consistency_score` (또는 `dci_score`)
    *   **타입:** Float (0.0 ~ 100.0)
    *   **설명:** 해당 데이터 항목(예: 특정 영상의 시각 자료 패키지)이 Designer가 정의한 DCI 기준(`Deep Blue/Neon Cyan System`)을 얼마나 충족하는지를 측정하는 점수. (측정 방법론은 별도 문서 참조 필요)
    *   **연결 로직:** 이 필드는 백엔드에서 시각 자료의 메타데이터와 연결되어야 하며, KPI 계산에 직접적으로 사용됩니다.

**B. KPI 연계 지표 (Efficiency Linkage)**
*   **필드명:** `kpi_impact_factor` (또는 `efficiency_multiplier`)
    *   **타입:** Float
    *   **설명:** 해당 데이터 항목이 최종 수익화 목표(KPI) 달성에 기여하는 효율성 계수. 이는 경험의 깊이(Wisdom)와 AI 효율성의 전환 정도를 수학적으로 모델링한 결과값입니다.
    *   **연결 로직:** `visual_consistency_score`와 연계되어, 시각적 품질이 실제 비즈니스 성과에 미치는 영향을 정량화합니다.

#### 2. API 명세서 반영 요구사항 (구현 지시)

개발팀에게 다음 사항을 명확히 전달해야 합니다.

1.  **필드 추가:** 위에서 정의된 `visual_consistency_score`와 `kpi_impact_factor` 필드를 모든 관련 엔티티(예: `Subscription`, `BundleOption`)에 즉시 추가하도록 지시합니다.
2.  **데이터 소스 명시:** 이 두 필드의 값이 어떤 외부 시스템(Designer의 비주얼 가이드라인, KPI 모델)으로부터 유래하는지를 데이터베이스 스키마 주석으로 명확히 명시하도록 요구합니다.
3.  **연계 로직 구현:** `kpi_impact_factor`가 최종 수익화 지표에 어떻게 기여하는지에 대한 수학적 인과관계(Cause-Effect Model)를 API 응답에 포함하도록 요청합니다.
