## 💎 상품 모듈별 Sales Page Mockup 및 CTA 배치 확정 (System Architect 포지셔닝)

**💡 비주얼 시스템 요약:**
*   **Primary Color (신뢰):** Deep Blue (`#0A1931`) - 배경, 본문 영역.
*   **Accent Color (액션/강조):** Neon (Electric Blue/Cyan 계열, `#00FFFF` 또는 `#00BFFF`) - CTA 버튼, 핵심 키워드 강조.
*   **Typography:** 고딕체 기반으로 명확성 극대화. 제목은 볼드하고 크고, 시스템 요소는 구조적으로 배치.

### 🖼️ Mockup 1: 섹션 1 (The Hook) - 심리적 딜레마와 권위 확립

**[Layout Concept]** 배경 전체를 Deep Blue로 설정하여 깊이감과 신뢰감을 극대화합니다. 대비되는 Neon 색상을 사용하여 핵심 메시지를 즉각적으로 시선 집중시킵니다.

| 요소 | 내용 (Writer 카피) | 디자인 적용 (Color/Style) | 레이아웃 지침 |
| :--- | :--- | :--- | :--- |
| **배경** | 전체 페이지 배경 | `#0A1931` (Deep Blue) | Full Screen |
| **Headline** | 당신의 아이디어는 실패하지 않았습니다. 하지만 당신의 시스템은 파산할 위험에 처해 있습니다. | 폰트: Bold Sans-Serif, 크기 최대 | 중앙 정렬, White/Light Blue 톤으로 강조 |
| **Sub-Headline** | 단순한 디자인이나 툴 사용법이 아닙니다. 저는 당신의 비즈니스를 **자동화된 수익 엔진**으로 전환하는 **시스템 설계 방법론**을 제공합니다. (System Architect) | 폰트: Regular, Deep Blue 계열 | Headline 하단에 배치, Neon 강조 |
| **Visual Cue Block** | *당신은 아이디어를 가졌지만, 실행 계획이 없습니다.* / *이제 **실행 가능한 시스템**으로 전환하세요.* | 배경색: Soft Dark Blue (`#152C40`) / 텍스트: Neon Cyan (`#00BFFF`) | 좌우 분할 또는 중앙 블록 배치. Flow를 시각적으로 표현하는 다이어그램(System Blueprint)을 배경에 희미하게 오버레이. |

**✅ CTA 배치:** 이 섹션에서는 직접적인 CTA보다 다음 섹션으로의 자연스러운 흐름을 유도합니다. (다음 섹션 시작 전, 스크롤 유도 역할)

### ⚙️ Mockup 2: 섹션 2 (The Dilemma) - 현재의 비효율성 지적

**[Layout Concept]** 문제 제기 영역에서는 논리적인 흐름(Flow)을 시각적으로 명확하게 보여주어, 독자가 '현재 상황의 비효율성'에 공감하도록 유도합니다.

| 요소 | 내용 (Writer 카피) | 디자인 적용 (Color/Style) | 레이아웃 지침 |
| :--- | :--- | :--- | :--- |
| **Title** | 기존 방식의 함정 | 폰트: Bold, Deep Blue | 좌측 상단 배치 |
| **Body Copy** | 수많은 툴과 파편화된 노하우는 당신을 **더 지치게** 만들 뿐입니다. 개별적인 작업은 당장의 결과를 줄지 몰라도, 확장성과 지속 가능성을 보장하지 못합니다. 비효율적인 워크플로우는 잠재적 수익을 갉아먹습니다. | Deep Blue 배경 위에 흰색/밝은 회색 본문. 핵심 단어(**더 지치게**, **확장성**)를 Neon으로 강조. | 좌측에 배치. 문제점을 나열하는 목록(Bullet Points) 형태로 구성. |
| **Visual Flow** | 파편화된 프로세스 (Fragmented Process) → 비효율적 자원 낭비 / 시스템 설계 (System Design) → 확장 가능한 수익 엔진 | 다이어그램 강조 | 중앙 또는 우측에 수평/순차적인 화살표 흐름(→)을 사용하여 명확한 **Before & After** 대비를 시각화. Fragmented Process는 회색 계열, System Design은 Neon Blue로 처리하여 대조를 극대화. |

**✅ CTA 배치:** 이 섹션은 독자의 문제 인식을 높이는 단계이므로, 직접적인 버튼보다는 다음 해결책(System 제시)으로의 전환을 유도하는 시각적 연결에 집중합니다.

### 🚀 Mockup 3: 섹션 3 (The Solution) - 핵심 가치 및 방법론 소개

**[Layout Concept]** 솔루션을 제시하는 영역이므로, 가장 명확하고 강력한 신뢰감을 주는 레이아웃을 사용하며, 최종 행동(CTA)을 극대화합니다.

| 요소 | 내용 (Writer 카피) | 디자인 적용 (Color/Style) | 레이아웃 지침 |
| :--- | :--- | :--- | :--- |
| **Title** | 아 빨랑~~ 시스템 설계 방법론 | 폰트: Bold, Neon Blue (`#00BFFF`) | 중앙 정렬. 가장 시선을 끄는 제목으로 설정. |
| **Core Value Proposition (3단계)** | 1. Foundation (기반) / 2. Implementation (구현) / 3. Transformation (변화) | 각 단계별 아이콘과 핵심 키워드 배치 | 세로 또는 수평의 명확한 3분할 구조. 각 단계는 박스 형태로 분리하여 시각적 위계를 부여. |
| **Step Details** | *Foundation:* 데이터베이스 스키마, 모듈 구조 정의... / *Implementation:* 시스템 모듈별 콘텐츠 및 액션... | 각 항목은 Deep Blue 배경의 카드(Card) 안에 배치. | 3분할 영역 내에 상세 설명으로 삽입. Foundation은 신뢰감을 주는 짙은 Blue, Implementation과 Transformation은 Neon 계열로 대비를 주어 실행력을 강조. |
| **Final CTA Block** | (최종적인 행동 유도 문구 필요) | 버튼: `지금 시스템 설계 시작하기` / 색상: Neon (`#00FFFF`) | 페이지 하단에 고정된(Sticky) 형태로 배치하여 이탈을 최소화. 배경은 대비되는 짙은 Blue 유지. |
