## 📊 최종 시각화 리소스 패키지 완성 프레임워크 (데이터 동기화 기반)

본 프레임워크는 Writer가 제공하는 최종 스크립트 및 Developer의 실시간 데이터를 기반으로, 정의된 **Deep Navy/Neon Cyan 시스템**을 유지하면서 실제 데이터가 채워졌을 때 적용할 수 있는 최종 시각화 리소스 패키지를 완성하기 위한 구조를 제시합니다.

### 1. 핵심 디자인 규칙 (Static System - 고정)

모든 동적 요소는 다음의 브랜드 가이드라인을 절대적으로 준수해야 합니다.

*   **배경 색상:** Deep Navy (`#0A192F`)
*   **강조/액션 색상 (Neon):** Neon Cyan (`#00FFFF`)
*   **타이포그래피 스타일:** 산세리프 계열, 미래지향적 느낌 강조.
*   **비주얼 리듬 규칙:** 3~5초 간격의 빠르고 명확한 전환(Transition)을 기본으로 하며, 정보 전달 시 Neon Cyan을 핵심 KPI에만 집중적으로 사용한다.

### 2. 데이터 동기화 매핑 테이블 (Dynamic Mapping Table)

Writer/Developer가 제공하는 DB 필드와 스크립트 흐름을 디자인 요소로 변환하는 규칙입니다.

| DB 데이터 항목 (예시) | 콘텐츠 역할 (스크립트 연관성) | 시각적 표현 방식 (Visual Element) | 색상 및 강조 지침 |
| :--- | :--- | :--- | :--- |
| **LTV** | 투자 가치 증명 (Why Invest) | 중앙 집중형 KPI 박스 | Neon Cyan (가장 높은 주목도) |
| **CAC** | 비용 효율성 분석 (Cost Analysis) | 대비되는 보조 바 또는 그래프 | Deep Navy 배경 위에서 회색 계열로 처리하여 대비 유지 |
| **ROI Ratio** | 최종 결과 및 수익성 | Dynamic Bar Chart 또는 Gauge Meter | Neon Cyan으로 채워진 진행률 표시 |
| **구독 기간** | 사용자의 참여/신뢰도 | 시간 흐름 시각화 (Timeline) | Deep Navy 배경 위에서 Neon Cyan 라인 강조 |
| **변동 데이터 (예: 월별 수익)** | 변화 추이 (Trend) | 미세한 움직임을 가진 Line Graph | Neon Cyan의 굵기(Stroke Width)를 데이터 값에 비례하여 동적으로 조정 |

### 3. 최종 시각화 리소스 패키지 구조 (Template Structure)

모든 영상 포스트 및 웹사이트 섹션은 아래의 모듈식 구조를 따릅니다. 이는 데이터 입력 시, 각 모듈이 자동으로 업데이트되도록 설계됩니다.

#### A. 도입부 (Hook & Credibility - 0-5초)
*   **레이아웃:** 중앙 집중형 타이포그래피와 배경 비주얼 강조.
*   **핵심 요소:** LTV/CAC 비율을 가장 크게 표시하며, **ROI Ratio**를 Neon Cyan으로 즉각적으로 제시. (데이터 입력 시, 이 수치가 시스템의 핵심 지표가 됨)

#### B. 문제 제기 및 솔루션 (Pain Point $\rightarrow$ Solution)
*   **레이아웃:** 좌측에 Pain Point 텍스트, 우측에 Solution 시각화 박스 분할.
*   **동적 적용:** 데이터의 논리적 흐름(Module 1, 2)에 따라 배경 색상의 미묘한 변화(Deep Navy $\rightarrow$ Darker Navy)를 사용하여 섹션 구분을 명확히 함.

#### C. 과정 및 증명 (Process & Proof - Transformation)
*   **레이아웃:** **동적 프로세스 다이어그램 템플릿** 적용.
*   **핵심 요소:** DB의 논리 구조(Module 1, 2, 3)에 따라 각 단계별로 Neon Cyan 화살표와 연결선이 동적으로 생성되어야 함. (예: Module 1 완료 $\rightarrow$ Module 2 시작 시, 전환 효과 발생)

#### D. 최종 결과 및 CTA (Result & Action)
*   **레이아웃:** 최종 ROI 수치를 가장 크게 배치하고, 하단에 명확한 CTA 박스를 배치.
*   **동적 적용:** **최종 ROI Ratio**가 100% 달성되었을 때, 전체 화면이 Neon Cyan의 강렬한 그라데이션으로 전환되는 시각적 피날레를 준비합니다.

### 4. 실행 지침 (Designer Action Plan)

Writer로부터 최종 스크립트와 데이터 구조(DB Schema)가 제공되면, 위 매핑 테이블과 패키지 구조에 따라 Figma 내에서 **'데이터 입력용 변수화 레이어'**를 구축하고 시뮬레이션을 즉시 생성합니다. 특히 C 섹션의 동적 다이어그램 템플릿을 최우선으로 완성합니다.
