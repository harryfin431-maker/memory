# 모듈 판매 시각 자료 패키지 마스터 구조 및 최종 요구사항

웹사이트 개발 지연 리스크에 선제적으로 대응하기 위해, $T_{dev}$ 수신 즉시 비주얼 에셋 생성이 차질 없이 진행될 수 있도록 '모듈 판매 시각 자료 패키지'의 마스터 구조와 최종 요구사항을 아래와 같이 확정하고 준비 상태를 유지합니다.

## 1. 비주얼 시스템 재확인 (Deep Blue/Neon 기반)

모든 모듈 판매 시각 자료는 기존에 확정된 브랜드 시스템을 엄격히 준수해야 합니다.

### 1.1. 컬러 팔레트 (Color Palette)
| 용도 | 색상 코드 (Hex) | 역할 |
| :--- | :--- | :--- |
| **Primary Blue** | `#0A2342` (Deep Blue) | 신뢰감, 배경 및 핵심 타이포그래피 |
| **Accent Neon** | `#00FFFF` (Neon Cyan) | 강조 요소, CTA 버튼, 하이라이트 |
| **Secondary Accent** | `#FF00FF` (Neon Magenta) | 보조 강조, 수익성 지표 시각화 |
| **Background** | `#FFFFFF` / `#F0F8FF` | 콘텐츠 영역 및 배경 |

### 1.2. 타이포그래피 가이드라인 (Typography Guidelines)
*   **Headline Font:** Montserrat Bold (신뢰감과 미래지향적 느낌 강조)
    *   *사용 예시:* 모듈 제목, 핵심 메시지
*   **Body Font:** Inter (가독성 극대화)
    *   *사용 예시:* 상세 설명, 텍스트 정보

## 2. 모듈 판매 시각 자료 마스터 구조 (Template Structure)

모든 모듈 판매 시각 자료는 다음의 3가지 표준 템플릿 중 하나를 따릅니다. 이는 개발 완료 후 즉시 적용 가능하도록 설계되었습니다.

### Template A: 수익성 지표 강조형 (KPI Focus)
*   **목표:** $Ad-ROI$ 및 성과 측정을 시각화하여 신뢰도 극대화.
*   **레이아웃 특징:** 좌측에 핵심 KPI(수익, 전환율 등)를 대형 Neon 색상으로 배치하고, 우측에 모듈의 기능 및 결과 설명을 배치.
*   **핵심 요소:** Deep Blue 배경 위에 Neon 컬러의 그래프/차트 시각화.

### Template B: 프로세스 흐름 강조형 (Process Flow Focus)
*   **목표:** 모듈 사용 과정을 단계별로 명확하게 제시하여 이해도를 높임.
*   **레이아웃 특징:** 수평 또는 수직으로 3~5단계의 프로세스를 화살표와 아이콘을 사용하여 연결.
*   **핵심 요소:** Deep Blue 배경에 White/Neon 텍스트를 활용한 단계별 설명 박스.

### Template C: 결과물 집중형 (Result Showcase Focus)
*   **목표:** 최종 결과물의 시각적 매력을 극대화하여 구매 욕구 자극.
*   **레이아웃 특징:** 중앙에 모듈의 핵심 스크린샷 또는 최종 결과물을 배치하고, 주변 여백을 충분히 확보하여 시선 집중.
*   **핵심 요소:** 고해상도 이미지 중심. Deep Blue 프레임 및 Neon 하이라이트 박스 사용.

## 3. 비주얼 에셋 생성 최종 요구사항 (Asset Generation Requirements)

개발팀으로부터 $T_{dev}$ 수신 즉시, 아래의 목록에 따라 모든 모듈 판매용 시각 자료(썸네일 포함)를 생성합니다.

1.  **썸네일 컨셉 3안 적용:** 이전에 확정된 썸네일 컨셉 3안(Session: sessions/2026-04-29T02-55/designer.md 참고) 중, **Template A와 Template C를 결합한 형태**로 우선 제작을 시작합니다. (수익성과 결과물 시각화의 균형 추구)
2.  **마스터 에셋 패키지 정의:** 각 템플릿에 필요한 모든 아이콘 세트(Neon 스타일), 그래프 템플릿, 배경 패턴(Deep Blue/Neon 그라데이션)을 FIGMA 내에 마스터 컴포넌트로 정의합니다. (재사용성 극대화)
3.  **텍스트 프레임워크 준비:** 각 템플릿에 들어갈 헤드라인, 본문 설명의 **Placeholder 텍스트 구조**를 미리 정의하여 디자인 작업 시간을 최소화합니다.
