### 📋 백엔드 API 명세서 및 실시간 연동 구현 계획 (초안)

#### 1. 데이터 모델 수정 요구사항 (API 명세 반영)

기존의 `Subscription`, `BundleOption`, `TransactionLog` 데이터 모델에 **시각적 일관성 지표(DCI)**와 **핵심 수익화 지표(KPI)**를 연결하는 필드를 추가해야 합니다. 이는 단순한 데이터 저장을 넘어, '경험의 깊이(Wisdom)'가 'AI 효율성(Efficiency)'으로 전환되는 인과관계를 측정 가능하게 만듭니다.

**✅ 필수 추가 필드 정의:**

| 테이블 | 필드명 | 데이터 타입 | 설명 | 연계 지표 (KPI/DCI) | 연계 로직 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `Subscription` | `dci_score` | Float | 해당 구독/상품의 시각적 일관성 점수 (0.0 ~ 1.0) | DCI | Designer가 정의한 Threshold 기준 충족 여부 |
| `BundleOption` | `kpi_impact_factor` | Float | 이 번들 옵션이 주요 수익 지표에 미치는 영향 계수 | KPI 연계 | 수학적 모델링 결과 (기존 데이터 기반) |
| `TransactionLog` | `efficiency_metric` | Float | 사용자 행동의 효율성 측정값 | AI 효율성 | 사용 시간 대비 전환율 등 계산된 값 |

**⚙️ API 엔드포인트 명세 반영:**

*   `/api/v1/subscriptions/{id}`: DCI 점수(`dci_score`)와 KPI 영향 계수(`kpi_impact_factor`)를 포함하여 반환하도록 수정.
*   `/api/v1/transactions`: `efficiency_metric` 필드를 포함하여 사용자 행동 로그를 기록 및 집계할 수 있도록 설계.

#### 2. 실시간 연동 구현 계획 (Data Pipeline & Logic)

데이터 모델이 확정되면, 이 데이터를 **실시간으로 반영하고 분석**하는 파이프라인을 구축해야 합니다.

*   **Phase 1: 데이터 수집 및 정규화 (Pipeline)**
    *   **목표:** DB에 저장된 DCI/KPI 지표를 실시간 대시보드(Dashboard)로 스트리밍하기 위한 ETL(Extract, Transform, Load) 파이프라인 구축.
    *   **기술 스택:** 기존 백엔드 API (Node.js/Python 등)의 데이터베이스 변경 사항을 감지하는 CDC(Change Data Capture) 메커니즘 도입 검토.

*   **Phase 2: 연계 로직 구현 (Logic Implementation)**
    *   **목표:** DCI 점수와 KPI 지표 간의 상관관계를 실시간으로 계산하여, 어떤 시각적 일관성이 수익화에 가장 큰 영향을 미치는지 분석하는 알고리즘을 백엔드 서비스 레이어에 통합.
    *   **핵심 로직:** $\text{Efficiency} = f(\text{DCI}, \text{Conversion Rate})$ 모델 적용.

*   **Phase 3: 프론트엔드 연동 (Visualization)**
    *   **목표:** 데이터 파이프라인을 통해 집계된 지표를 `dashboard-kit`의 차트 컴포넌트에 실시간으로 매핑하여 시각화. DCI 점수가 낮을수록 KPI 달성률이 하락하는 추이를 즉시 확인할 수 있도록 설계.
